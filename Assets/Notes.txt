    эта команда выполняется

Нам нуден интерфейс который будет реализовывать исполнитель команд этот интерфейс содержит один метод, который будет принимать инткрфейс команды
ICommandExecutor
    CommandExecute(object command)

CommandExecutorBase<T>
    
    public void CommandExecute(object command) => ExecutSpecificCommand((T)command)
    public abstract void ExecutSpecificCommand(T command){}

CommandButtonsPresenter - класс презентер, соединяет модель selectableValue и view - CommandButtonView класс который отвечает за отрисовку и вывод кнопок команд
В старте происходит подписка на событие модели и когда в модели происходит изменение текущего выделения, то презентер реагирует на методом onSelected
который занимается отрисовкиой команд выделенного объекта. В методе происходит получение всех компонентов интерфейсов которые отвечают за выполнение команд.
Они помещаются в Лист далее вызывается метод вью который принимает вкачестве параметра лист содержащий интерфейсы команд и отрисовывает UI кнопки команд.

Далее нужно подписать на собития нажатия кнопок вызовы соответствующих команд исполнителяю Для этого заведем метод onButtonClick который в старте будет подписан сообзен с делегатом класса commandButtonView
который звнимается подпиской.

Создаем атрибут, которым будем помечать поля в классе не monobehavior.

Создадим папку Utils в ней создадим одноименный файл assembly defenition. В этой папке создадим подкоталок AssetInjector и в нем создадим
атрибут.
Для этого создадим новый класс отнаследуемый от System.Attribut
Укажем, какие элементы данного типа помечать аттрибутом. Имя класса аттрибута должно заканчиваться словом attribut в последствии оно обрежиться средой

Мы хотим, чтобы наша утилита установил ассет с именем Enemy01 в поле помеченное атрибутом.

Прежде чем проставлять значения в поля, мы создадим объект контекст, который будет работать с ассетами.
В нашем случае это будет класс scriptableObject
Этот класс будет содержать список наших префабов и иметь один метод который будет возвращать объект по типу и имени
Для внедрения зависимости нам нужно:
    Класс в который будем внедрять зависимость
    Атрибут которым будет помечено поле для внедрения зависимости
    Класс контекст, который будет работать с объектами для внедрения
